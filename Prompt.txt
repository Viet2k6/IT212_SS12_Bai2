Prompt 1:
# Vai trò
Bạn là một Senior Business Analyst kiêm System Analyst có hơn 10 năm kinh nghiệm trong lĩnh vực FinTech và Ngân hàng số.
Bạn có kinh nghiệm viết Software Requirements Specification (SRS) theo chuẩn IEEE 830/IEEE 29148.

# Mục tiêu
Tôi sẽ cung cấp tài liệu Business Analysis của hệ thống mở tài khoản trực tuyến (eKYC).
Hãy ghi nhớ toàn bộ ngữ cảnh để sử dụng ở các Prompt tiếp theo.
Chưa cần sinh tài liệu.
Chỉ trả lời:
"Đã tiếp nhận ngữ cảnh dự án và sẵn sàng xây dựng SRS."

Prompt 2:
Dựa trên tài liệu Business Analysis đã ghi nhớ, hãy viết Phần 1 của tài liệu SRS.
Yêu cầu trình bày theo chuẩn IEEE.
Bao gồm:
# 1. Introduction
## 1.1 Purpose
## 1.2 Scope
## 1.3 Definitions
## 1.4 Acronyms
## 1.5 References
Viết theo văn phong tài liệu kỹ thuật chuyên nghiệp.
Xuất kết quả bằng Markdown.

Prompt 3:
Tiếp tục xây dựng tài liệu SRS.
Viết Phần 2: Overall Description.
Bao gồm:
## Product Perspective
## Product Functions
## User Classes and Characteristics
## Operating Environment
## Design Constraints
## Assumptions and Dependencies
Trình bày đầy đủ theo chuẩn IEEE.
Xuất Markdown.

Prompt 4:
Tiếp tục tài liệu SRS.
Viết Phần 3: Specific Functional Requirements.
Phân tích chi tiết các module sau:
1. Đăng ký tài khoản
2. Upload CCCD
3. OCR đọc dữ liệu CCCD
4. Liveness Check
5. Face Matching
6. Đối chiếu dữ liệu
7. Kích hoạt tài khoản
Đối với mỗi module hãy trình bày: Mục tiêu, Actors, Preconditions, Main Flow, Alternative Flow, Exception Flow, Postconditions, Business Rules, Validation Rules.
Trình bày chuyên nghiệp dưới dạng Markdown.

Prompt 5:
Tiếp tục tài liệu SRS.
Viết Phần 4: Non Functional Requirements.
Bao gồm đầy đủ:
Security, Performance, Availability, Reliability, Scalability, Maintainability, Logging, Audit, Backup & Recovery, Compliance.
Đối với Performance hãy nêu rõ: Response Time, Concurrent Users, API Timeout.
Đối với Security hãy nêu: Encryption, Authentication, Authorization, Data Privacy, Audit Log.
Xuất Markdown.

Prompt 6:
Tiếp tục tài liệu SRS.
Sinh mã Mermaid cho Use Case Diagram.
Yêu cầu:
Actors: Customer, eKYC System
Use Cases: Register, Upload ID Card, OCR Processing, Liveness Check, Face Matching, Validate Information, Generate Account Number, Activate Account.
Chỉ trả về code Mermaid.
